# 🔒 Политика безопасности

## Сообщение об уязвимостях

Если вы обнаружили уязвимость безопасности, пожалуйста, сообщите нам ответственно:

### Как сообщить

1. **Не создавайте публичный Issue** — это может подвергнуть пользователей риску
2. **Отправьте email** на security@example.com
3. **Или используйте GitHub Security Advisories** (если включено)

### Что включать

- Описание уязвимости
- Шаги для воспроизведения
- Доказательство концепции (если возможно)
- Потенциальное воздействие
- Предлагаемое исправление (если есть)

### Чего ожидать

- Ответ в течение 48 часов
- Обновление статуса в течение 7 дней
- Исправление в течение 30 дней (для критичных)
- Публичное уведомление после исправления

## Лучшие практики

### Для пользователей

- Всегда используйте последнюю версию
- Регулярно обновляйте зависимости
- Не коммитьте `.env` файлы с секретами
- Используйте сложные пароли
- Включите 2FA (когда доступно)

### Для разработчиков

- Проверяйте зависимости на уязвимости
- Не логируйте чувствительные данные
- Используйте параметризованные запросы
- Валидируйте все входные данные
- Следуйте принципу наименьших привилегий
